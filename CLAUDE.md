# Issue駆動型自律開発システム

## プロジェクト概要

GitHub Issue をタスクキューとし、ラベル付与をトリガーに Claude Code が自律的に実装→PR作成を行うシステム。人間の指示がボトルネックにならず、非同期でレビュー・修正サイクルを回せる。

## アーキテクチャ

```
[人間] ── Issue起票 + ラベル付与 ──> [GitHub Actions]
                                          │
                                          v
                              [claude-code-action@v1]
                              (.claude/ のスキル・エージェント・ルールを読み込み)
                                  │              │
                                  v              v
                            [ブランチ作成]    [Issue コメント]
                            [実装 + テスト]   [質問・報告]
                            [PR 作成]
                                  │
                                  v
                          [人間がPRレビュー]
                          [@claude コメント で修正指示]
                                  │
                                  v
                            [人間がマージ]
```

## 自律実装のルール

### トリガー

- Issue に `claude:implement` ラベルを付与すると自律実装を開始
- PR に `@claude` コメントで修正指示を送ると対応

### 実装時のルール

1. **ブランチ戦略**: `issue-{number}` ブランチを作成し、main には直接コミットしない
2. **テスト必須**: 実装には対応するテストを含める
3. **PR作成**: 実装完了後に PR を作成し、`Closes #{number}` で Issue を紐付ける
4. **確認事項**: 判断に迷った点は PR の「確認事項」セクションに記載する
5. **質問**: 不明点が多い場合は Issue にコメントで質問し、`claude:waiting` ラベルを付与する

### 利用可能なスキル・エージェント

`.claude/skills/` および `.claude/agents/` に定義されたスキル・エージェントを活用する。主要なもの:

- `/implement` — タスク実装
- `/test` — テスト実行
- `/build-check` — ビルド・Lint チェック
- `/code-review` — コードレビュー

## ドキュメント体系

`docs/` 配下のドキュメント体系については `.claude/CLAUDE.md` を参照。

## 開発環境
