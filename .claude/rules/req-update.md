# 要件変更・追加の検知

ユーザーが以下のような発言をした場合、要件変更・追加の可能性があると判断し、対話の中で `/req-update` スキルの使用を提案する:

- 仕様・要件の変更について言及した場合（「〜を変えたい」「〜の仕様を修正」「〜は不要になった」など）
- 新しい要件・機能の追加について言及した場合（「〜も追加したい」「〜の機能が欲しい」など）
- 既存の仕様に対する疑問・再検討を示した場合（「〜はこれでいいのか」「〜を見直したい」など）

提案の仕方:
1. まずユーザーの意図を対話で確認する
2. 変更/追加の内容が固まったら、`/req-update <内容>` の実行を提案する
3. ユーザーが直接 `/req-update` を実行した場合はそのまま進める
