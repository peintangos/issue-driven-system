# steering/ 作業ログ規約

## ディレクトリ命名
`YYYY-MM-DD-<slug>/`（例: `2026-02-22-add-link-card/`）

## 必須ファイル
1. `plan.md` — 目的・やること・影響範囲。以下を必ず含める:
   - **対象 Specification**: `docs/specifications/` のどの仕様のどの実装ステップか
   - **全体ロードマップにおける位置づけ**: `docs/roadmap.md` との整合性を確認
2. `tasklist.md` — タスク一覧とチェックボックスで進捗管理
3. `blocker.md` — ブロッカー・課題（なければ「なし」と記載）
4. `knowledge.md` — 学び・気づき（作業後に追記）
