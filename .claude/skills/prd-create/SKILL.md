# PRD作成（初回MVP）

## 概要

新規プロジェクトの初回MVP PRDを対話的に作成し、specificationsへの分割・roadmap/progressの初期化まで行う。

## 手順

### 1. 現状把握

以下を確認する:
- プロジェクトの目的・背景（ユーザーとの対話で確認）
- 既存のドキュメント（`docs/` 配下）

### 2. ユーザーとの対話

以下のトピックについてユーザーと議論する:
- プロダクトのビジョンと目的
- ターゲットユーザー（ペルソナ）
- MVP に含める機能の優先順位
- 技術的な制約条件
- UX 要件

### 3. PRD作成

議論の結果を `docs/prd.md` に記載する。

### PRD テンプレート

```markdown
# プロダクト要件定義書（PRD）— MVP

## 概要

[このプロダクトで実現すること]

## 背景

[なぜこのプロダクトが必要か]

## プロダクト・プリンシプル（プロダクト原則）

[設計原則・価値観]

## スコープ

### 含むもの
- [機能1]
- [機能2]

### 含まないもの
- [スコープ外の機能]

## 対象ユーザー

[ペルソナ・ターゲットユーザー]

## ユースケース

[主要なユースケース]

## 機能要求

[具体的な機能要件]

## UX要求

[ユーザー体験に関する要件]

## システム要求

[技術的・システム的な要件]

## マイルストーン

| マイルストーン | 内容 | 目標時期 |
|-------------|------|---------|
| ... | ... | ... |
```

### 任意セクション（必要に応じて追加）

- 市場分析
- 競合分析
- セキュリティ要件
- プライバシー要件
- パフォーマンス要件
- マーケティング計画

### 4. Specificationsへの分割

PRDの機能要求をもとに、ユーザーと相談しながら個別のspecificationに分割する:
- 各specificationは `/spec-create` で作成するよう案内する
- 機能の粒度・優先順位をユーザーと確認する

### 6. README.md の更新
- プロジェクト名と概要を README.md に記載する
- 詳細は `docs/prd.md` を参照するリンクを含める

### 5. Roadmap / Progress 初期化

- `docs/roadmap.md` にMVPの構想・計画を記載する
- `docs/progress.md` にMVPセクションを作成し、各specificationの進捗欄を追加する

### 7. todo 更新

- `docs/todo/todo.md` にマイルストーンから導出されるタスクを追加する
- 各specificationの実装ステップなど、詳細が多い場合は詳細ページ（`docs/todo/<slug>.md`）を作成してリンクする

## 注意事項

- 日本語で記述する
- ユーザーとの対話を重視し、一方的にドラフトを作らない
- 全てのセクションを一度に決める必要はなく、段階的に埋めてよい
- このスキルではソースコードの実装・変更は行わない。ドキュメントの更新のみに留める
