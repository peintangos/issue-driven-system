ユーザーからのラフな依頼をもとに、GitHub issueを起票してください。

## 手順

1. **内容の整形**: ユーザーの依頼（`$ARGUMENTS` や会話の文脈）から、以下を構成する
   - **タイトル**: 簡潔で明確（日本語OK）
   - **本文**: 背景・やりたいこと・受け入れ条件などをMarkdownで整形

2. **ユーザーに確認**: 起票前に、整形した内容を表示して確認を取る
   ```
   📋 以下の内容でissueを起票します：

   タイトル: [title]
   ラベル: [labels if any]

   本文:
   [body]

   よろしいですか？
   ```

3. **起票**: 承認されたら `gh issue create` で起票する

## issueテンプレート

本文は以下の構成を基本とする（内容に応じて柔軟に調整）：

```markdown
## 概要
[何をしたいか / 何が問題か]

## 詳細
[背景や具体的な内容]

## 受け入れ条件
- [ ] [完了条件1]
- [ ] [完了条件2]
```

## ルール

- ユーザーの言語に合わせる（日本語で依頼されたら日本語で）
- ラフな依頼でも意図を汲み取って適切に整形する
- 不明点があれば起票前に質問する
- ラベルやアサインはユーザーが指定した場合のみ付与する
- 起票後はissueのURLを表示する
